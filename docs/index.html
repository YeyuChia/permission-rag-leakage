Permission-Aware RAG Leakage Demo

This project reproduces cross-permission information leakage in a multi-user enterprise RAG setup. It is inspired by access control challenges discussed in AC-LORA (NeurIPS 2025).

Repository

Source code: github.com/YeyuChia/permission-rag-leakage

Key Idea

  • secure retrieval filters documents by user permission
  • vulnerable retrieval searches all zones (missing ACL filter)

Permission Zones

ZoneExample Sensitive Content
publicCompany overview
project_alphaProject budget ($2.3M)
confidentialQ3 layoff plan, executive salary

Demo Commands

python -m src "What is Project Alpha's total approved budget for 2025?" --user guest --mode vulnerable
python -m src.eval
streamlit run demo/app.py

Expected Result

In vulnerable mode, a guest user can retrieve Project Alpha / confidential documents and trigger LEAK DETECTED. In secure mode, leakage rate should drop significantly.
